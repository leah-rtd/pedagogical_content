Introduction la Rgression Linaire en Machine Learning

La Rgression Linaire

La rgression linaire cherche ajuster une droite qui reprsente au mieux la relation entre une variable explicative ($x$) et une variable cible ($y$) :
$y = ax + b$

O :

  • $a$ est la pente de la droite
  • $b$ est l'ordonne l'origine

L'objectif est de minimiser l'erreur entre les valeurs prdites par le modle et les valeurs relles observes.

Comment faire la cross validation

Mthode train_test_split (aussi appele Hold Out Method)

In[6]:
from sklearn.model_selection import train_test_split


# Sparer nos features et target en jeu d'entrainement et jeu de test
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size = 0.3)

# Instancier le modle
model = LinearRegression()

# Entrainement du modle
model.fit(X_train, y_train)

# Evaluation du modle sur le jeu de test
model.score(X_test, y_test)
Out[6]:
0.6017954898635374

Par dfaut, le score pour la mthode model.score() pour une tache de rgression est le $R^2$, mais ce score n'est pas trs reprsentatif.
Nous allons utiliser la Root Mean Squared Error (RMSE) ou racine de l'erreur quadratique moyenne. Cette mtrqiue reprsente l'cart-type des rsidus et pnalise plus largement les grandes erreurs.

In[7]:
from sklearn.metrics import root_mean_squared_error

score = root_mean_squared_error(y_true = y_test, y_pred = model.predict(X_test))
score

La target $y$ est le prix mdian des maisons dans un California block group, ce prix est en centaines de milliers de dollars (100,000 USD).
Le RMSE de notre modle est donc de 73,141 USD

La mthode train_test_split n'est pas assez robuste. Nous voulons avoir une ide de la performance gnrale d'un modle qui a vu toute la donne
